# 06 — UI/UX System

## Design Goal

SuperNova should feel like a professional operations control system.

It should not look like:

```text
Generic HR SaaS
Toy dashboard
Random template
Decorative landing page
```

Target feel:

```text
Branch-first workflow tool
Role-based operational workspace
Clean admin control center
Clear request/approval timeline
Mobile-first field operations UI
```

## Required UI Skill

For UI/UX work, use:

```text
ui-ux-pro-max
```

If available, use it to:

```text
Audit current page
Plan layout
Review responsive behavior
Improve hierarchy
Review forms and workflow clarity
Polish visual details
```

If unavailable, manually apply this document.

Do not use UI skill usage as permission to change backend logic.

## Mobile-First Standard

Design for:

```text
360px
390px
430px
```

Every mobile-critical page must avoid:

```text
horizontal overflow
tiny buttons
unclear primary action
large desktop-only tables
crowded forms
random decorative cards
```

## Component Rules

Use:

```text
Cards for grouped context
Tables for dense admin lists
Badges for statuses
Timelines for workflows
Side panels or bottom sheets for current action
Searchable selectors for large lists
```

Do not show giant dropdowns for all branches/chains.

Use search for large operational entities.

## Form Rules

Workflow forms must include:

```text
Entity context
Required fields
Validation messages
Approval path preview
Review before submit
Clear submit button
Safe cancel/back action
```

## Request Detail Rules

Request detail must show:

```text
Request type
Status
Current required action
Creator
Target user/candidate
Source Branch/Chain
Destination Branch/Chain if applicable
Approval steps
Timeline
Final result summary
```

## New Hire UI Direction

Unified New Hire form modes:

```text
Branch-locked mode
Global search mode
Organization shortcut mode
```

Must support:

```text
targetRole selection based on user permission
Chain search
Branch search scoped by selected Chain and role
Candidate lookup after valid phone/national ID
Read-only previous user card
Blocked user warning card
New candidate data form when no old data exists
Approval path preview
Final review
```

## Copy Style

Use operational copy.

Good:

```text
Selected Branch
Approval path
Pending Admin Finalization
Open User Profile
Temporary password available in profile
```

Bad:

```text
Empower your team
Magic workforce experience
Click here
```

## Visual Rules

Use:

```text
neutral background
white cards
soft borders
orange as controlled brand accent
consistent status colors
clear spacing
large touch targets
```

Do not overuse color.

## Verification

Every UI task must state:

```text
Mobile widths checked
Desktop checked
No horizontal overflow
Primary action visible
Loading state checked
Error state checked
Empty state checked
```
