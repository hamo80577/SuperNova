# UI/UX Direction

## Product Feel

SuperNova should feel like a professional operations control system.

It should not look like:

```text
generic HR SaaS
marketing landing page
toy dashboard
decorative template
```

Target feel:

```text
branch-first workflow tool
role-based operational workspace
clean admin control center
clear request/approval timeline
mobile-first field operations UI
```

## Mobile-First Standard

Design first for:

```text
360px
390px
430px
```

Avoid:

```text
horizontal overflow
tiny buttons
large desktop-only tables
crowded forms
unclear primary action
random decorative cards
fake data
```

## Attendance Operations UX Direction

The future Attendance Data Operations page should feel like an operational import console.

It should prioritize:

```text
clear period selection
obvious upload mode
preflight summary
processing progress
issue visibility
final import outcome
safe destructive operation flow
```

Do not use marketing copy.

Use direct operational labels such as:

```text
Upload Attendance
Import History
Data Maintenance
Calculation Rules
Ignored Non-Egypt Rows
Unmatched Identifiers
Ambiguous Matches
Rebuilt Summaries
```

## Verification Expectations for UI Phases

Future UI phases must verify:

```text
mobile widths 360/390/430
desktop layout
no horizontal overflow
primary action visible
loading state
error state
empty state
long table/list behavior
danger confirmation flow
```
